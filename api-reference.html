Authentication

DockEase supports two authentication methods:

Bearer JWT (Session-based)

Obtain a JWT pair by calling POST /api/v1/auth/login/. Include the access token in all subsequent requests:

Authorization: Bearer <access_token>

Access tokens expire after 15 minutes. Use the refresh endpoint to obtain a new access token using a valid refresh token. Refresh tokens expire after 7 days.

API Key (Pro/Enterprise)

Generate an API key from your account settings. Pass the key as a header:

X-API-Key: dk_live_xxxxxxxxxxxxxxxxxxxxxxxx

API keys do not expire but can be revoked from the dashboard. They carry the same permissions as the owning user account.

Rate Limiting

All endpoints are rate-limited. Exceeding limits returns HTTP 429 Too Many Requests with a Retry-After header.

ScopeLimitWindow
Login (POST /auth/login/)5 requests1 minute
Password reset (POST /auth/password/reset/)5 requests1 hour
Anonymous (unauthenticated)60 requests1 minute
Authenticated user600 requests1 minute
API key (Pro)600 requests1 minute
API key (Enterprise)3000 requests1 minute

Pagination

List endpoints return paginated results in the following format:

json{
  "count": 142,
  "next": "https://api.dockease.io/api/v1/jobs/?page=3",
  "previous": "https://api.dockease.io/api/v1/jobs/?page=1",
  "results": [ ... ]
}

Pagination parameters:

ParameterTypeDefaultDescription
pageinteger1Page number (1-indexed)
page_sizeinteger20Results per page (max: 100)

Error Format

All errors return a consistent JSON structure:

json{
  "error": {
    "code": "RECEPTOR_NOT_FOUND",
    "message": "Receptor with id 'abc123' does not exist or you do not have permission to access it.",
    "details": {}
  }
}

Common HTTP Status Codes

CodeMeaning
200 OKSuccessful GET, PATCH
201 CreatedSuccessful POST
204 No ContentSuccessful DELETE
400 Bad RequestValidation error — check error.details
401 UnauthorizedMissing or invalid authentication
403 ForbiddenAuthenticated but insufficient permissions
404 Not FoundResource does not exist
409 ConflictState conflict (e.g., deleting a running job)
429 Too Many RequestsRate limit exceeded
500 Internal Server ErrorServer error — contact support

Auth Endpoints

POST /api/v1/auth/register/

Create a new user account.

Request Body:

json{
  "email": "researcher@university.edu",
  "password": "SecurePassphrase123!",
  "full_name": "Dr. Jane Smith",
  "affiliation": "University of Cambridge"
}

Response 201:

json{
  "id": "usr_01HN8K3DFQX",
  "email": "researcher@university.edu",
  "full_name": "Dr. Jane Smith",
  "email_verified": false,
  "created_at": "2026-03-14T10:23:41Z"
}

Error Codes:

  • EMAIL_ALREADY_EXISTS (400) — An account with this email already exists
  • WEAK_PASSWORD (400) — Password does not meet complexity requirements

POST /api/v1/auth/login/

Authenticate and receive JWT tokens.

Request Body:

json{
  "email": "researcher@university.edu",
  "password": "SecurePassphrase123!"
}

Response 200:

json{
  "access": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refresh": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": "usr_01HN8K3DFQX",
    "email": "researcher@university.edu",
    "full_name": "Dr. Jane Smith",
    "plan": "pro",
    "jobs_used_this_month": 47,
    "jobs_limit": 500
  }
}

Error Codes:

  • INVALID_CREDENTIALS (401)
  • EMAIL_NOT_VERIFIED (403)
  • ACCOUNT_SUSPENDED (403)

curl example:

bashcurl -X POST https://api.dockease.io/api/v1/auth/login/ \
  -H "Content-Type: application/json" \
  -d '{"email":"researcher@university.edu","password":"SecurePassphrase123!"}'

Python example:

pythonimport requests

resp = requests.post(
    "https://api.dockease.io/api/v1/auth/login/",
    json={"email": "researcher@university.edu", "password": "SecurePassphrase123!"}
)
tokens = resp.json()
access_token = tokens["access"]

POST /api/v1/ligands/from-smiles/

Create a ligand from a SMILES string.

Request Body:

json{
  "smiles": "CC(=O)Oc1ccccc1C(=O)O",
  "name": "Aspirin",
  "project_id": "proj_01HN9X"
}

Python example:

pythonimport requests

headers = {"Authorization": f"Bearer {access_token}"}
resp = requests.post(
    "https://api.dockease.io/api/v1/ligands/from-smiles/",
    json={
        "smiles": "CC(=O)Oc1ccccc1C(=O)O",
        "name": "Aspirin",
        "project_id": "proj_01HN9X"
    },
    headers=headers
)
ligand = resp.json()
print(f"Ligand ID: {ligand['id']}, MW: {ligand['molecular_weight']}")

WebSocket Protocol

DockEase uses WebSockets to stream real-time job progress.

Connection

wss://api.dockease.io/ws/jobs/{job_id}/?token={access_token}

Authentication is via the JWT access token as a query parameter. API key authentication is not supported on WebSocket connections.

Connection Lifecycle

Client → Server: Connect
Server → Client: {"type": "connection_established", "job_id": "job_01HQ2X"}
Server → Client: {"type": "job_progress", ...}  (repeated)
Server → Client: {"type": "job_completed", ...}  (on completion)
Client → Server: Close

Message Types

connection_established

json{
  "type": "connection_established",
  "job_id": "job_01HQ2X",
  "current_status": "running",
  "current_progress": 45
}

job_progress

Sent approximately every 2 seconds while the job is running.

json{
  "type": "job_progress",
  "job_id": "job_01HQ2X",
  "status": "running",
  "progress_percent": 67,
  "ligands_completed": 6,
  "ligands_total": 9,
  "current_ligand": "compound_007",
  "elapsed_seconds": 84,
  "eta_seconds": 41,
  "timestamp": "2026-03-14T11:32:25Z"
}

job_completed

json{
  "type": "job_completed",
  "job_id": "job_01HQ2X",
  "status": "completed",
  "result_count": 81,
  "best_affinity_kcal_mol": -10.4,
  "duration_seconds": 162,
  "timestamp": "2026-03-14T11:33:42Z"
}

job_failed

json{
  "type": "job_failed",
  "job_id": "job_01HQ2X",
  "status": "failed",
  "error_code": "LIGAND_PREP_FAILED",
  "error_message": "Ligand compound_003 could not be prepared: invalid valence.",
  "timestamp": "2026-03-14T11:32:10Z"
}

Reconnection

The DockEase frontend implements exponential backoff reconnection:

  • Initial reconnect delay: 1 second
  • Backoff multiplier: 2×
  • Maximum delay: 30 seconds
  • Maximum attempts: 10

If implementing your own WebSocket client, we recommend adopting the same strategy to handle transient network interruptions without overwhelming the server.

Python WebSocket example:

pythonimport asyncio
import websockets
import json

async def watch_job(job_id: str, access_token: str):
    url = f"wss://api.dockease.io/ws/jobs/{job_id}/?token={access_token}"
    async with websockets.connect(url) as ws:
        async for message in ws:
            data = json.loads(message)
            if data["type"] == "job_progress":
                print(f"Progress: {data['progress_percent']}% ({data['ligands_completed']}/{data['ligands_total']})")
            elif data["type"] == "job_completed":
                print(f"Done! Best affinity: {data['best_affinity_kcal_mol']} kcal/mol")
                break
            elif data["type"] == "job_failed":
                print(f"Failed: {data['error_message']}")
                break

asyncio.run(watch_job("job_01HQ2X", "eyJ..."))
